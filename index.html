System Architecture

Adaptive Syllabus Pipeline

The fully automated engine: from real-time JD scraping to module deployment and student outcome measurement. This is the operational logic underpinning the VAIL-OEH Ecosystem.

🔄 Autonomous Operating Engine — 6-Stage Cycle
RUNNING
🌐
Stage 1 · Market Data Scanning
Octoparse continuously collects 10,000+ JDs/day from VietnamWorks, LinkedIn, TopCV. Keywords: "marketing", "digital", "AI". Extracts: required skills, tools, qualifications.
→
🧮
Stage 2 · NLP Vectorization
Python NLTK/spaCy tokenizes JD text. TF-IDF vectors built for each skill mention. Cosine similarity computed against syllabus document vectors. Output: similarity matrix.
→
📊
Stage 3 · Gap Detection & Heatmap
Similarity index calculated per skill cluster × university. 35 missing skills identified, grouped into 8 clusters. Heatmap auto-generated. Scores below 0.60 flagged.
↓
📈
Stage 6 · Impact Measurement
Pre/Post test scores compared. Improvement % calculated per student. Urban vs rural cohort analysis via MGA. Survey tracks self-reported skill confidence. Dashboard updated in real-time.
←
🎓
Stage 5 · Student Learning
18 pilot students (9 urban, 9 rural) access module. Pre-test → 15-min AI-narrated video → Knowledge summary → Post-test → Survey. All tracked anonymously.
←
🚀
Stage 4 · Module Auto-Deployment
AI Curriculum Co-Pilot drafts micro-learning content based on flagged skill gaps. Module pushed to faculty every Monday. Students notified. Content: video + summary + quiz + survey.
